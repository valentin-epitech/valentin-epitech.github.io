The EntitiesStructures class

This class regroups all the structures you will be receiving and sending to your clients, you will find players, lasers, and enemies's infos.

The ENTITY_TYPE enum

This enum is always read first when the server receives an information by the client to know what type of entity (structure) the client is sending him. If it's a PLAYER, it will be read like a player, if it's a LASER, like a laser, etc...

The GAME_PHASE enum

This enum is used by the server to know which enemies to handle and at what point, since each wave is involves new mecanics. It is never send or receive by anyone.

The t_player structure

This contains the player's informations. You will find it's type PLAYER, his Health, his ID, if he's ready to start the game or not (isReady), and his x and y 2D positions.

The t_laser structure

This contains the lasers's informations. You will find it's type LASER, which player it's attached to (playerID), his ID (LaserID), and his x and y 2D positions.

The t_hostile_laser structure

This contains the informations of the enemy's lasers. You will find it's type HOSTILE_LASER, and it's x and y 2D positions.

The t_hostile_laser structure

This contains the informations of the enemy's lasers. You will find it's type HOSTILE_LASER, and it's x and y 2D positions.

The t_ball_entity structure

This contains the informations of the ball that follows the player. You will find it's x and y 2D positions and ID (linked to the player's ID).

The t_enemy_1 & t_enemy_2 structures

They are essentially the same and contains the enemies's informations. You will find it's x and y 2D positions, Health, ID, and type. The t_enemy_1 is for the first wave, and the t_enemy_2 for the second wave.

The t_final_boss structure

Contains the final boss's informations. You will find it's x and y 2D positions, Health, ID, and type. He's just like a normal enemy, but with higher stats.

The EnemiesFunctions class

This class is made to handle the mecanics of the enemies. Movements, lasers, death, etc...

The up_or_down enum

Since the bots are moving verticaly, this is made to know in which direction they're going.

First exchange between client and server

The client connect -> The server sends the client his ID

Entity exchange between client and server

The client sends an Entity, regardless which one -> The server answer by sending all the structures in a precise order.
